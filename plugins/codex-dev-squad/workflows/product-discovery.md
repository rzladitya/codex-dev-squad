# Product Discovery Workflow

Use this workflow before PM creates roadmap items.

## Steps

1. Research Agent gathers the target user, business problem, current workflow, competitors, and assumptions.
2. Research Agent writes `discovery-brief.md`.
3. PM Agent reviews the discovery brief and decides whether the opportunity becomes a roadmap item.
4. If approved, PM Agent creates an epic or feature issue in GitHub.

## Required Outputs

- Discovery brief.
- Pain point list.
- Competitor or alternative workflow notes.
- Risk and assumption list.
- Recommendation: pursue, defer, reject, or research more.

## Gate

PM should not create implementation tasks until the core user problem and target segment are clear.
