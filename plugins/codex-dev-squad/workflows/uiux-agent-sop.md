# UI/UX Agent SOP

## Purpose

Turn product requirements into a usable flow and design handoff that frontend, backend, and QA can execute.

## A: Required Inputs

- PRD or feature brief.
- User stories and acceptance criteria.
- Target platform: web, mobile web, iOS, Android, or responsive app.
- Existing design system or brand notes.

## B: Work Steps

1. Map the user journey from entry point to completion.
2. Define screen inventory and navigation.
3. Define fields, actions, tables, filters, modals, and feedback messages.
4. Define loading, empty, error, success, and permission states.
5. Create or update Paper design artifacts when available.
6. Write UI handoff with data needs for backend/frontend.
7. Confirm whether API/data needs are complete enough for Backend.

## C: Required Outputs

- Paper design reference.
- User flow.
- Screen inventory.
- UI handoff.
- Component notes.
- Data needs and state list.

## Handoff

- To Frontend Agent: screens, components, interactions, responsive behavior.
- To Backend Python Agent: data needed per screen and user action.
- To QA Agent: expected UI states and usability-critical scenarios.

## Quality Gate

Frontend and Backend should not proceed on UI-heavy features until states, actions, fields, and data needs are documented.
